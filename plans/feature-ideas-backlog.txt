ocraft — feature ideas backlog
==============================
(generated 2026-06-19)

A wide brainstorm of NEW feature ideas, beyond the 17 README roadmap items and the
8 in project-direction-and-ideas.txt. One line each; grounded in ocraft's real
primitives (nodes, parentId, the `x` context, the kernel API, the scheduler, MCP,
the migrated notes). This is a backlog to PRUNE, not a commitment. `[ext]` = sharpens
an existing roadmap/analysis item. Promote the best into the README roadmap.


1. NODE MODEL & PRIMITIVES
--------------------------
1.  Typed relations/edges beyond parentId — named links (references, depends-on, blocks); the tree becomes a graph.
2.  Cross-cutting tags/labels — orthogonal to the tree, for filter/group (#project, #idea).
3.  Node aliases / symlinks — one node appears in multiple tree locations without duplication.
4.  Structured properties per type — typed key/value fields on nodes (Notion-like), queryable.
5.  Multiple bodies/attachments per node — e.g. a script + its README + tests in one node.
6.  Node transclusion/embed — render one node inside another (block include).
7.  Computed/derived nodes — content is the live result of a script, auto-refreshing.
8.  Type inheritance/mixins — "meeting" extends "html" with extra fields.
9.  Persistent trash + restore — beyond the undo pool, a real trash category.
10. Node color/icon/emoji — lightweight visual identity in the tree.

2. THE `x` RUNTIME (node-as-program)
------------------------------------
11. `x.store` stdlib — create/save/delete/query/tree from inside a script [ext keystone].
12. `x.state` — persistent per-node key/value across runs (scripts remember).
13. `x.emit`/`x.on` — local node-to-node event bus (later bridged to ws-exchange).
14. `x.schedule` — a node registers itself/another as a scheduled task from inside.
15. `x.fetch` — authenticated, CORS-proxied outbound HTTP using the x.auth token store.
16. Named exports per node — `x.x("5",{fn:"foo"})` calls a named export, not just default.
17. Typed node contracts — declare input/output schema; x.x calls validated + autocompleted.
18. `x.render` — formalize returning a renderable the handler displays. ✅ DONE as "view" script
    nodes (node.view = vue-esm | vue-sfc): default export `(x)=>Component` mounts in the editor;
    vue-sfc compiles a real .vue (template + script setup + auto-scoped <style>) at runtime via a
    lazy, code-split compiler; one shared Vue instance. NEXT (roadmap): on-the-fly `<style lang=scss>`
    (add sass) + `<script setup lang=ts>` compilation for vue-sfc; later React/Svelte/Solid renderers
    (same (hostEl, component) → cleanup mount seam in useScriptView.js).
19. Multi-language WASM bodies — Rust/Zig/C → WASM nodes; or run an uploaded .wasm node.
20. `pipeline` node type — declaratively chain node calls (A→B→C) as visual dataflow.
21. Node imports — a script declares dependencies on other node ids, resolved before run.
22. Script↔service bridge — promote a long-running script node into a supervised service.
