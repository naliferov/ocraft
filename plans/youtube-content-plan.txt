WHY THIS PROJECT IS GOOD YOUTUBE MATERIAL
-----------------------------------------
ocraft is unusually demoable because it spans four "expertise lanes" at once,
and almost everything has a VISUAL or AGENTIC payoff you can record:
  - FULL-STACK CRAFT — a from-scratch Vue 3 + p5.js visual editor and a
    zero-framework Node HTTP/API/scheduler backend (no Express, no Nest). "I
    built the framework" reads as senior signal.
  - AGENTIC AI — Claude Agent SDK embedded in the app (the ai-chat node), a fleet
    of custom MCP servers (Telegram/GCal/Gmail/DigitalOcean), a Dockerized "yolo"
    agent runner, and a self-improving dev loop. This is the hot lane right now.
  - CREATIVE CODING — p5.js scenes, post-process effects, Tone.js audio, WASM
    (AssemblyScript) in the browser, a toy language interpreter. Eye/ear candy.
  - INFRA / SYSTEMS — cron scheduler with PID-file locking, a process supervisor,
    Deno Deploy edge WebSocket relay, image pipelines. "It actually runs in prod."

POSITIONING (pick one, stay consistent):
  A. "Building my own life-OS in public" — narrative, relatable, series-friendly.
  B. "Senior-level engineering, shown not told" — deep-dives that prove depth.
  C. Hybrid: a build-in-public SERIES (A) with occasional deep-dive specials (B).
  RECOMMENDED: C. The series gives cadence + story; the deep-dives are the ones
  that get shared by other engineers and become portfolio links.

FORMATS:
  - FLAGSHIP (10-20 min) — one feature, real build/demo, architecture diagram.
  - DEEP-DIVE (20-40 min) — the hard parts; code on screen; trade-offs.
  - SHORT (<60s) — one "whoa" moment (drag-to-nest, AI edits its own editor, WASM
    speedup, voice-note → summary). Shorts are the discovery engine; cut them FROM
    the flagship footage, don't film twice.
  - Every flagship should ship with 1-3 shorts carved out of it.

PRODUCTION NOTES:
  - The editor IS the set — screen-record the running app; the claude-in-chrome
    MCP / screenshots already give you "eyes" for clean capture.
  - Keep a reusable cold-open: 5s of the flashiest result, then "here's how."
  - Show real code, but cut to the diff that matters; don't scroll dead files.
  - SECURITY: never show .env, the root `token` file, real API tokens, the
    Telegram session, or private notes on screen. The ai-chat endpoint is
    bypassPermissions/localhost — say so, don't expose it.

--- TIER 1: FLAGSHIPS (shoot these first — highest signal, most shareable) ---

1) "I gave my own app an AI that can edit itself" — the ai-chat node
   SHOW: type a request into the in-app chat node; Claude Agent SDK (cwd = repo)
     reads/edits files and runs commands; the editor changes live. Show tool_use
     events streaming, the cost/turns result.
   PROVES: embedding the Claude Agent SDK, agent loops, tool-use streaming, and
     the security model (bypassPermissions, localhost-only, no API key — uses your
     Claude login). Senior judgement = you explain WHY it's dangerous + sandboxed.
   HOOK: "My app can rewrite its own source while I watch."
   EFFORT: medium. MONEY: this is the SaaS seed — "agentic editor" is a product.
     MARKETING: the self-modifying demo is a perfect short + HN/X bait.

2) "I built a visual node editor from scratch (Vue 3 + p5.js)" — the editor core
   SHOW: the typed-node model (scene/script/html/category/ai-chat), JSON-on-disk
     store, recursive tree, the p5 render loop, sidecar bodies (script.js /
     content.html kept out of state.json).
   PROVES: front-end architecture, reactivity (Pinia), recursive components,
     canvas/p5 rendering, clean data modeling. "No framework did this for me."
   HOOK: "Notion-meets-game-engine, and I wrote the engine."
   EFFORT: medium. MONEY: the editor is the core product surface. MARKETING:
     gorgeous to watch; the architecture diagram is a great thumbnail.

3) "Drag-to-nest, undo, and a tree that's actually a filesystem" — CRUD + nesting
   SHOW: create/rename/delete nodes, drag a node onto a `category` to reparent
     (with the cycle guard refusing a node into its own subtree), collapse/expand,
     "Recently deleted" restore, the 409-on-orphan backend rule.
   PROVES: HTML5 drag-and-drop done right (custom drag ghost!), tree algorithms,
     optimistic UI + persistence, careful delete semantics (no silent orphaning).
   HOOK: "Building a file tree is harder than it looks — here's every trap."
   EFFORT: low-medium (already built — just narrate + record). MONEY: table-stakes
     UX. MARKETING: the drag-ghost detail is a satisfying short.

4) "Custom MCP servers: giving Claude hands on my real life" — MCP fleet
   SHOW: Claude using the Telegram (read/search your own account via MTProto),
     Google Calendar, Gmail, and DigitalOcean MCPs — e.g. "spin up a droplet",
     "what's on my calendar", "summarize this chat". Show the thin server.js +
     shared *-api.js pattern.
   PROVES: the Model Context Protocol end-to-end, tool schema design (zod),
     auth/token handling, API integration, and a reusable house style across
     servers. This is THE portfolio piece for the agent era.
   HOOK: "I wrote the tools that let an AI run my infrastructure."
   EFFORT: medium. MONEY: MCP consulting/templates; a paid "MCP starter kit".
     MARKETING: a series — one episode per MCP. DigitalOcean (real droplet
     created live) is the jaw-dropper.

5) "Real-time at the edge: a WebSocket relay on Deno Deploy" — the WS exchange
   SHOW: the Hono/Deno relay (ws-exchange), the in-editor websocket-tester node,
     two browser tabs talking through stream.x8.deno.net live; then the planned
     token gate + the deno-deploy MCP deploying it.
   PROVES: WebSockets, pub/sub relay design, edge/serverless deploy, Deno vs Node,
     and (with the plan) deploy automation + auth. Polyglot range.
   HOOK: "Multiplayer, served from the edge, deployed by an AI."
   EFFORT: medium-high. MONEY: real-time backend is a hostable service.
     MARKETING: the two-tabs-chatting demo is instantly legible.

--- TIER 2: STRONG DEEP-DIVES (engineer-to-engineer, high share among devs) ---

6) "WASM in the browser, compiled on demand" — AssemblyScript nodes
   SHOW: write AssemblyScript in a script node; server compiles it to wasm via
     `/api/nodes/:id/wasm` (lazy asc import); the browser instantiates + calls it;
     benchmark vs JS for a hot loop.
   PROVES: WASM toolchain, AssemblyScript, lazy server compilation, JS↔wasm
     interop, performance benchmarking. Rare, senior, very shareable.
   HOOK: "I type TypeScript, the server hands back WebAssembly."
   EFFORT: medium. MONEY: low direct. MARKETING: the speedup graph is the short.

7) "Scripts that call other scripts — composition inside the editor"
   SHOW: a node script invoking another by id (`x.x("5", args)`), the cache +
     cache-bust-on-save, the `x.ui` controls surface (buttons/inputs/log panel).
   PROVES: a tiny runtime/DSL, dynamic blob imports, dependency caching,
     building a mini "stdlib" (x.ui). Language/runtime design taste.
   HOOK: "I turned nodes into callable functions."
   EFFORT: low-medium. MONEY: power-user feature → paid tier. MARKETING: short
     showing one slider driving another node.

8) "A cron scheduler and a process supervisor, no libraries" — backend engine
   SHOW: jobs[] (interval + cron-like + activeHours), the executor + execution
     log, PID-file locking (withLock), procManager supervising procs with
     rotating logs and start/stop/restart/status.
   PROVES: systems programming, concurrency control (locks), process lifecycle,
     log rotation, durable JSON state. "I reimplemented the boring-but-hard parts."
   HOOK: "cron + pm2 + a job runner, in plain Node — and why I didn't use a lib."
   EFFORT: medium. MONEY: low direct. MARKETING: dev-credibility content; great
     for "show your fundamentals" audience.

9) "Voice note → text → action" — Telegram + whisper.cpp pipeline
   SHOW: pull a Telegram voice note via the MCP, ffmpeg → 16k mono wav, whisper
     -cli (large-v3-turbo) transcribe (multilingual!), then summarize/act with AI.
   PROVES: media pipelines, local ML inference (whisper.cpp), shell orchestration,
     multi-tool chaining. Practical + impressive.
   HOOK: "I talk to my phone; my computer files it."
   EFFORT: low-medium. MONEY: "voice-to-task" is a sellable micro-product.
     MARKETING: hyper-relatable short.

10) "I turned my Obsidian vault into a queryable node tree" — ThinkTank migration
   SHOW: the one-time importer (notes → html nodes, wikilinks → /node/:id, images
     optimized + served), then browsing/searching the migrated tree in the editor.
   PROVES: data migration, link rewriting, content modeling, idempotency/caveats
     (why re-running is destructive), image optimization (sharp).
   HOOK: "My notes are now a database I can automate against."
   EFFORT: medium. MONEY: "import your vault" onboarding for a notes product.
     MARKETING: the Obsidian community is a ready audience.

--- TIER 3: NARRATIVE / VISION (binge-able series + creative flair) ---

11) "Building a personal life-OS in public" — the umbrella series
   SHOW: the vision — ThinkTank as memory, scheduler as the when/what, MCPs as
     I/O, Claude as judgment. Each later video = one organ of the system.
   PROVES: product thinking, systems architecture, follow-through.
   HOOK: "One repo to run my whole life. Episode 1." EFFORT: low (talking + b-roll).
   MONEY: the series itself (audience → product funnel). MARKETING: the spine that
     ties every other video together; strongest subscribe driver.

12) "The daily briefing that writes itself" — the daily-report skill
   SHOW: one command → unread Gmail + this week's Calendar + ThinkTank todos + a
     Telegram tech-channel digest, consolidated.
   PROVES: multi-source aggregation, skill/automation design, summarization.
   HOOK: "My morning report, generated while I sleep." EFFORT: low.
   MONEY: a "personal briefing" subscription product. MARKETING: everyone wants
     this — high relatability short.

13) "Letting an AI safely run wild: Dockerized yolo agent" — claude-yolo.js
   SHOW: running Claude Code in a sandboxed Docker container (full autonomy, but
     contained); why isolation matters; what it can/can't touch.
   PROVES: containerization, sandboxing, the safety story for autonomous agents.
   HOOK: "Full-autonomy AI, in a box that can't hurt me." EFFORT: medium.
   MONEY: "safe agent runner" is a product/consulting angle. MARKETING: the
     safety framing stands out amid reckless-agent hype.

14) "The app that improves itself" — autonomous dev loop (when built)
   SHOW (once shipped): a plan skill writes a sequenced plan; a dev-task entry
     executes ONE task per run on a branch via the Agent SDK; opens a PR; never
     touches main. Show a real AI-authored PR.
   PROVES: agentic software engineering, CI-of-AI, guardrails (PR-only, human
     review, no auto-merge). Frontier content.
   HOOK: "I went to bed; my repo had a pull request waiting." EFFORT: high (needs
     the feature built first — see the README Roadmap). MONEY: this is a startup. MARKETING:
     one of the most viral possible angles if it genuinely works.

15) "Creative coding nodes: p5 scenes, grain, and Tone.js" — generative art
   SHOW: scene nodes (viewport letterbox, elements, the grain pixel-effect),
     tempo/structure music metadata, Tone.js audio; the "Claude as performer"
     idea (render-and-return loops).
   PROVES: graphics, audio synthesis, real-time rendering, creative/algorithmic
     range.
   HOOK: "Code that you watch and hear." EFFORT: medium. MONEY: paid asset/effect
     packs; creative-coding course. MARKETING: pure eye/ear candy → Shorts/Reels
     gold.

16) "I wrote a tiny programming language" — the `yo` interpreter (js-engine)
   SHOW: yo's lexer → parser → evaluator, the `@`-keyword async-first syntax,
     run program.yo; pitch it as a future sandboxed node-scripting medium.
   PROVES: language implementation — the ultimate "I understand computers" flex.
   HOOK: "Lexer, parser, evaluator — from zero." EFFORT: medium-high.
   MONEY: low direct. MARKETING: language-nerd catnip; strong dev-cred piece.
