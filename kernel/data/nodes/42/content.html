Immutability. In FP data is immutable; functions process data by returning new values instead of mutating it.

Side effects. In functional programming people try to minimize side effects and strive for pure functions, which always return the same result for the same input data and have no visible side effects.

Functional programming usually relies heavily on recursion for iteration and data processing, whereas procedural programming uses loops and iterative constructs.

In functional programming, functions can be passed as arguments to other functions or returned from functions. This allows writing more abstract and flexible code.
