треды vs cluster node:cluster — плохой способ масштабирования при > 6–8 процессов: все подключения проходят через родительский процесс по IPC, главный процесс на 100%, дочерние недогружены

Лучше: 1 порт на тред, балансировка в отдельном Auth-сервисе. Клиент при первом контакте получает токен и несколько точек подключения host:port, подключается по токену к своему треду. При разрыве - параллельные попытки к основной и запасным точкам. Весь трафик не через один балансировщик - распределение только при создании сессий

Передача CPU-жрущих операций в воркер дешевле, чем в другой инстанс: через SharedArrayBuffer и MessagePort, с CAS — через расшаренную память.

ECONNRESET — обработка

process.on('uncaughtException', (error, origin) => {
  if (error?.code === 'ECONNRESET') return;
  console.error('UNCAUGHT EXCEPTION');
  console.error(error);
  console.error(origin);
  process.exit(1);
});

Standarts implementation:

  • Node.js 0.x – 4.x → в основном ECMAScript 5 (ES5), но некоторые фичи ES6 можно было включить через --harmony.
  • Node.js 6.x → появилась частичная поддержка ECMAScript 6 (ES6), такие вещи как let, const, стрелочные функции, Map, Set, Promise.
  • Node.js 8.x → async/await (из ECMAScript 2017).
  • Node.js 10.x → почти полный ES6+ и новые фишки вроде optional catch binding (из ES2019).
  • Node.js 12.x → экспериментально поддерживает ESModules (import/export).
  • Node.js 14.x → ESModules стали стабильными, а также добавили поддержку ECMAScript 2020 (?., ??, BigInt).
  • Node.js 16.x → новые возможности ES2021 (String.prototype.replaceAll, ??=, ||=, &&=).
  • Node.js 18.x → поддержка ES2022 (Array.prototype.at, Error.cause).
  • Node.js 20.x+ → уже включает ES2023 и частично ES2024.
