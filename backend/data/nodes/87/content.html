PostgreSQL 15 (2022) добавил долгожданный стандартный оператор MERGE для upsert-логики, JSON-log формат для логов и улучшил логическую репликацию (фильтрацию по таблицам/колонкам)​ supabase.

Также ускорились сортировка и сжатие WAL (поддержка zstd-сжатия)​, aws, что важно для высоконагруженных систем. PostgreSQL 16 (2023) продолжил тренд: повысил производительность параллельных запросов, дал возможность создавать статистики для выражений, что улучшает планирование сложных запросов. В сообществе Postgres развиваются расширения – например, pgvector для хранения эмбеддингов (поддерживая вышеупомянутый AI-тренд) и pg_partman для удобной партиционирования данных по времени. Популярность Postgres привела к появлению многочисленных облачных сервисов на его основе: Amazon Aurora PostgreSQL, Google AlloyDB, Neon и др., предлагающих масштабирование, бессерверный режим и мгновенные бэкапы. Отдельно стоит отметить проект Neon – это облачная PostgreSQL, разделяющая хранение и вычисления, позволяющая быстро «замораживать» и «возобновлять» БД (использует S3 как хранилище и compute-нод по запросу). Также в 2023 Amazon представил Babelfish for Postgres – слой совместимости, позволяющий Postgres понимать протокол T-SQL/MS-SQL, что облегчает миграцию с Microsoft SQL Server.

Индексы PostgreSQL

Типы: B-tree (по умолчанию), hash, GiST, SP-GiST, GIN, BRIN. Также: BITMap, covering index. Полнотекстовый поиск: SELECT * FROM articles WHERE MATCH(title, content) AGAINST('запрос');

EXPLAIN смотрит статистику из ANALYZE TABLE, а не реальное кол-во строк.

ACID

  • Atomicity: операция выполняется полностью или не выполняется вообще
  • Consistency: после операции БД переходит в корректное состояние
  • Isolation: параллельные операции не влияют друг на друга
  • Durability: изменения сохраняются даже при сбоях системы

CRDT (Conflict-free Replicated Data Type)

Структуры данных для распределённых систем без конфликтов при репликации. Типы: счётчики, множества (sets), карты (maps), списки (lists) с timestamp, деревья с ID+timestamp.

SQL EXAMPLES

CREATE TABLE objects (
id SERIAL PRIMARY KEY,
data JSONB NOT NULL,
previous_id INTEGER REFERENCES objects(id) ON DELETE SET NULL,
next_id INTEGER REFERENCES objects(id) ON DELETE SET NULL
);

CREATE TABLE objects_operations (
id SERIAL PRIMARY KEY,
data JSONB NOT NULL,
);

CREATE TABLE projects (
id SERIAL PRIMARY KEY,
name VARCHAR(255) NOT NULL,
user_id INTEGER NOT NULL REFERENCES users(id)
);

CREATE TABLE project_object (
project_id INTEGER REFERENCES projects(id) ON DELETE CASCADE,
object_id INTEGER REFERENCES objects(id) ON DELETE CASCADE,
PRIMARY KEY (project_id, object_id)
);

CREATE TABLE users (
id SERIAL PRIMARY KEY,
name VARCHAR(255) NOT NULL,
email VARCHAR(25) NOT NULL,
phash VARCHAR(64) NOT NULL
);
