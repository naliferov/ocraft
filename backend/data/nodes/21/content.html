**Authentication — who are you? Verifying identity. Authorization — what can you do? Verifying permissions.

Authentication methods

  • Password-based
  • Token-based (JWT, opaque tokens)
  • OAuth 2.0 / OpenID Connect
  • API keys
  • Certificate-based (mTLS)
  • Multi-factor (MFA): something you know + have + are

JWT (JSON Web Token)

Structure: header.payload.signature Stateless — server doesn't store session Signed (HS256, RS256) — tamper-proof Can be verified without DB lookup Expiry via exp claim

OAuth 2.0 flows

  • Authorization Code — web apps (most secure)
  • PKCE — public clients (SPA, mobile)
  • Client Credentials — machine-to-machine
  • Implicit — deprecated

Session vs Token

Session Token (JWT)
Storage Server-side Client-side
Revocation Easy Hard (need blocklist)
Scalability Requires shared store Stateless

Authorization models

  • RBAC — Role-Based Access Control
  • ABAC — Attribute-Based Access Control
  • ACL — Access Control List

Common vulnerabilities

  • Broken authentication (OWASP #2)
  • Token leakage (XSS, logs)
  • JWT alg: none attack
  • CSRF (session-based apps)
  • Insecure token storage (localStorage vs httpOnly cookie)

JWT (access) + Refresh tokens + Cookies This is the basic model. Specifically: access JWT — 5–10 minutes refresh token — 7–30 days

refresh is stored as: HttpOnly Secure SameSite=Strict | Lax

refresh tokens: stored on the server with rotation bound to the device / session
