Composition API

  • setup(), ref, reactive, computed, watch
  • Логика группируется по функциональности, а не по секциям data/methods
  • Удобный реюз через composables
  • Максимально дружелюбен к TypeScript

Новая система реактивности (Proxy)

  • Основана на Proxy, а не Object.defineProperty
  • ❌ Больше не нужен Vue.set / this.$set
  • Любые новые свойства становятся реактивными автоматически

Улучшенная поддержка TypeScript

  • Полная типизация компонентов
  • defineComponent, defineProps, defineEmits
  • Предсказуемые типы для props, emits, slots

fragments Несколько корневых элементов в шаблоне без обёртки <template> <span>A</span> <span>B</span> </template>

5. Teleport

Рендер компонента вне текущего DOM-дерева

<teleport to="body">
	<Modal />
</teleport>

Хорошо для:

  • модалок
  • тултипов
  • дропдаунов

6. Suspense

Асинхронные компоненты с fallback UI <Suspense> <template #default> <AsyncComponent /> </template> <template #fallback> Loading... </template> </Suspense>

7. Новый v-model

  • Поддержка нескольких v-model
  • Кастомные имена <MyInput v-model:title="title" />

Связка:

  • modelValue
  • update:modelValue

8. Производительность

  • Быстрее рендер
  • Меньше лишних перерисовок
  • Tree-shaking → меньше бандл

9. Новые lifecycle хуки

Вместо старых: beforeCreate, created

Используются: onMounted() onUpdated() onUnmounted() onActivated() onDeactivated()

10. <script setup>

Новый синтаксис SFC: <script setup> import { ref } from 'vue' const count = ref(0) </script>

  • Меньше шаблонного кода
  • Автоэкспорт
  • Быстрее компиляция
